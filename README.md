# Modular Token Tray 3d model generator

1×1×1 の Magnet Tray (Storage Box) の 3MF/STL を元に、任意サイズの
トレイと蓋の STL を生成するツールです。CLI と PyQt5 GUI の両方を含みます。

```
1モジュール      = 38.1 mm (1.5")
高さ1単位の増分  = 25.4 mm (1")   H1=22.2 / H2=47.6 / H3=73.0 mm ...
磁石             = φ5 x 2 mm ネオジム磁石 (穴は φ5.31 x 深さ2.03 mm)
```

## ファイル構成

| ファイル | 内容 |
|---|---|
| `magnet_tray_generator.py` | 生成エンジン + CLI |
| `magnet_tray_gui.py` | PyQt5 GUI (3Dプレビュー付き) |
| `test_magnet_tray.py` | 自動テスト (単体 / 生成マトリクス / 回帰) |
| `requirements.txt` | 依存パッケージ |
| `1x1x1_Storage_Box.3mf` | 元モデル (別途用意。GUIと同じフォルダに置くと自動読み込み) |

## インストール

Python 3.10 以降を推奨します。

```bash
pip install -r requirements.txt
```

`pyqtgraph` / `PyOpenGL` は GUI の高速 3D プレビュー用で任意です。
未インストールの場合は matplotlib プレビュー (視点ボタン切替) に
自動フォールバックします。

## 使い方

### GUI

```bash
python magnet_tray_gui.py
```

- 起動時に「前回使ったパス → スクリプトと同じフォルダ → カレントフォルダ」の
  順で 1x1x1 モデル (`*1x1x1*.3mf` → `*.3mf` → `box.stl`) を自動読み込みします。
  見つからない場合はオレンジの警告が表示されるので「参照...」で指定してください。
- サイズ (W/L/H)・内部区画・仕切り厚を設定して「プレビュー生成」→
  問題なければ「STL 保存...」。
- プレビュー操作 (OpenGL版): 左ドラッグ=回転 / ホイール=ズーム / 中ドラッグ=移動。
- ウィンドウ位置・元モデルパス・保存先は次回起動時に復元されます。

### CLI

```bash
# 3MF から 2x3、高さ2 を生成
python magnet_tray_generator.py --source 1x1x1_Storage_Box.3mf -W 2 -L 3 -H 2

# 抽出済み STL から、3x3 を 9区画に仕切って生成
python magnet_tray_generator.py --box box.stl -W 3 -L 3 -S 3,3 --out-dir ./out

# トレイのみ (蓋なし)、仕切り厚 2mm
python magnet_tray_generator.py --box box.stl -W 4 -L 2 -S 4,1 \
    --divider-thickness 2.0 --no-lid
```

| オプション | 説明 |
|---|---|
| `--source PATH` | 元の 1x1x1 の 3MF (Bambu Studio 形式対応) |
| `--box PATH` | 元トレイの STL (`--source` の代わり) |
| `-W / --width N` | 横モジュール数 |
| `-L / --length N` | 縦モジュール数 |
| `-H / --height N` | 高さ単位数 (+25.4mm/段。側壁磁石穴も段ごとに追加) |
| `-S / --sections A,B` | 内部区画数 「幅方向,奥行方向」(例: `2,1`)。デフォルト `1,1` |
| `--divider-thickness T` | 仕切り厚 mm (デフォルト 3.175 = 壁厚と同じ) |
| `--no-lid` | 蓋を生成しない |
| `--out-dir DIR` | 出力先 (デフォルト: カレント) |

出力ファイル名: `B_{W}x{L}x{H}.stl` (区画指定時は `_S{A}x{B}` が付く)、
蓋は `T_{W}x{L}.stl`。

## サイズ上限の目安 (Bambu Lab X2D)

X2D の造形サイズ 256 x 256 x 260 mm に基づく上限です
(GUI のスピンボックス上限もこの値):

| パラメータ | 上限 | 実寸 |
|---|---|---|
| W / L | 6 | 228.6 mm (7 モジュール = 266.7 mm は不可) |
| H | 10 | 250.8 mm |

CLI は上限チェックを行わないため、より大きな機種では任意の値を指定できます。

## 生成物の仕様

- **側壁磁石穴**: 各壁のモジュール中心 x 高さ 25.4mm 刻みの全位置
  (合計 `2*(W+L)*H` 箇所)。φ5.31 x 深さ 2.03 mm。
- **底面磁石穴**: 対角の 2 隅 (角から 4.57 mm インセット)。
- **磁石ポスト**: 逆対角の 2 隅。上面は常にリムの 2.03 mm 下
  (磁石を載せるとリムと面一になり、90°回した蓋を磁石で保持)。
- **蓋**: ハニカム構造 (ピッチ 7.94 mm)、厚さ 3.175 mm、
  磁石凹み 2 箇所 (トレイ底穴と同位置)。
- **仕切り**: 内寸を等分割する垂直リブ。床からリム上端まで
  (高さ方向の仕切りはなし)。磁石関連には影響しません。
- **必要磁石数**: トレイ `2*(W+L)*H + 4` 個、蓋 2 個 (5x2mm)。

1x1 の蓋・トレイと組み合わせても磁石位置は常に整合します
(コーナー系フィーチャーは「角から一定インセット」で配置されるため、
例えば 1x1x1 を 4 個並べた集合体に 2x2 の蓋がそのまま使えます)。

## 生成手法 (概要)

単純なスケーリングでは壁厚や磁石穴径が崩れるため、次の方式を採っています。

1. 元メッシュを「断面が位置不変」なカット面で 2 分割
   (XY: ±3.3 mm — 側壁磁石穴とポスト根元ブレンドを回避 /
   Z: +4.5 mm — 側壁穴上端とリム形状の間)
2. カット断面ポリゴンを正確に押し出したプリズムを挟んで拡張
   (フィレット・リム形状・ポストを完全保持。ポストは高さ拡張で自動的に伸長)
3. 不足する磁石穴をブーリアン減算で追加、仕切りをブーリアン加算

実行時には次の検証を行い、想定と異なる入力・出力はエラーで停止します。

- 元メッシュ: 寸法・原点位置・上下/回転の向き・磁石穴の特徴位置
  (反転・回転・原点ずれした STL を検出)、断面不変性
- 生成メッシュ: watertight / 面の向き / 有効なソリッド / 体積が正 /
  単一ボディ (分離ボディや内部空洞の混入を検出)

## テスト

```bash
python test_magnet_tray.py      # スタンドアロン実行
pytest test_magnet_tray.py -v   # pytest でも実行可
```

単体テスト (モジュール座標・寸法計算)、代表寸法の生成マトリクス
(1x1x1 〜 2x3x2、健全性・寸法・磁石穴数の検証)、回帰テスト
(W=L=H=1 の生成物が元モデルと体積差 0.001mm³ 未満で一致すること) を
含みます。元モデルをスクリプトと同じフォルダに置いて実行してください。

## トラブルシューティング

- **「元トレイの寸法が想定外です」**: ベースは 38.1 x 38.1 x 22.225 mm の
  1x1x1 (高さ1) モデルのみ対応です。
- **GUI のプレビューが表示されない/遅い**: `pip install pyqtgraph PyOpenGL`
  で OpenGL ビューが有効になります。
- **ブーリアンでエラー**: `pip install -U manifold3d trimesh` で更新して
  ください。

## ライセンス / 注意

- スクリプト (`*.py`) は自由に改変・利用してください。
- 元の 1x1x1 モデルおよび生成された STL は、元モデルの配布ライセンスに
  従います。生成物を配布する場合は元モデルの利用条件を確認してください。
- 印刷・磁石接着は自己責任で。磁石は極性を揃えて接着してください
  (隣接トレイの連結面同士が引き合う向き)。
